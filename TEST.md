# Evaluación de Conocimientos: GitHub y Markdown
 
## 1. Conceptos Generales
**¿Qué es un archivo Markdown y para qué sirve?**
> Un archivo Markdown es un archivo de texto plano que utiliza una sintaxis sencilla para dar formato al contenido (como títulos, listas, enlaces o negritas). Sirve para escribir documentos de forma clara y rápida, que luego pueden convertirse en HTML u otros formatos visuales.
 
**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**
*El archivo README.md sirve como la presentación principal de un repositorio, donde se explica de qué trata el proyecto, cómo usarlo, cómo instalarlo y otra información importante para los usuarios o desarrolladores.*

## 3. Flujo de Trabajo en GitHub Desktop
**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**
* **Commit:** Es guardar los cambios realizados en el repositorio de forma local, creando un registro con un mensaje descriptivo.
* **Push:** Es enviar los commits realizados en el repositorio local hacia el repositorio remoto en GitHub.
 
**¿Para qué sirve el botón "Fetch Origin"?**
> El botón "Fetch Origin" sirve para comprobar si hay cambios nuevos en el repositorio remoto sin descargarlos automáticamente, permitiendo ver si hay actualizaciones disponibles.
